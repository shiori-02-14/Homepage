みんなホームページを作ろう

こんにちは、あるいはこんばんは。

しおり🔖です。

突然ですが、皆さん、ホームページ作りませんか？

この記事を書いた一番の理由は、みんながどんなホームページを持ってるか、見てみたいから、って感じです。テンプレでも一からでも、作り方で見え方は全然違うし、個人サイトってどこか人の個性が出るところが好きです。

自分も SNS だけだと、自分の作ったものや学んだことをまとめて置いておくのがむずかしいな、と感じることが前々からあります。

自分がこのホームページを作った理由も、だいたいそれです。就活用のポートフォリオとかじゃなくて、自分があとから辿れる置き場が欲しかった、くらいです。

なので「みんなも作ってくれたらうれしい」くらいのノリです。必須でもないし、ブログだけで足りる人も、noteやQiitaだけで足りる人も多いと思います。

下の方に就活やインターンの話もありますが、自分は普段から働きたくないって言ってる側なので、そこは世の中のよくある言い方の整理にとどめてます。本筋は個性とかモチベとか技術の話です。

公開先を決めると、仕上げたくなることもあります。デザインは後からいじれるので、とりあえず適当でも出しちゃってもいいと思います。

なぜ個人サイトを持つとよいか

ここでいう個人サイト（ホームページ）は、だいたい自分の名前や活動がわかるページのことです。静的でもブログでもノート公開でも、なんでもありです。

個人サイトだからといって、技術系の記事を絶対書かなきゃいけないみたいな決まりはないです。エンジニアのポートフォリオみたいなイメージがつきやすいですが、日記や創作・写真・リンク集だけ、でも個人のページとして成立する気がします。

インターン・就活で「使える」という話

就活やインターンで使うときの話です。

作品やスキルが一発で辿れる

履歴書に URL を一行足すだけで、「詳しくはこちら」みたいな感じにできます。面接で話す内容と、実際に見せられるものがあると、説得力にもなるかな、と思います。

技術力のアピールにもなりやすい

書類に書いたスタックや、自分で組んだページ・デプロイの仕方・リポジトリへのリンクなどが並んでいると、「言っていること」と「見えるもの」がつながりやすいです。

すごい実装でなくても、自分の手で公開までやっていること自体が話の材料になることもあります。

学習や制作の続きが見える

更新日や記事の並びで、継続してきたかが伝わりやすいです。

志望や自己PRと整合しやすい

人事やったことないし完全に素人意見ですけど、ES に書いた技術スタックや興味と、サイトの中身がちゃんと揃っていると印象はよくなるのではないでしょうか。

しかし、あくまで書類や面接の補助線であって、「サイトがある＝評価される」わけでもないと思います。

作る・続けるモチベにつながりやすい理由

公開先があると書きやすい、という話。

公開する場所を先に決める

場所をとりあえず作っておくと、載せるために作品や記事などを整えたくなります。 自分はあるとモチベ上がるタイプ。

作品一覧や更新履歴がある

あとから見返したり、続きを書きやすかったりします。人によって効き方は違うかな、と思います。

気にしすぎかも、なやつ

「こうでなきゃ」みたいに思いがちな話です。自分もデザインも更新も、理想どおりにはできてないですが、あくまでよく言われる不安への整理です。

デザインが豪華じゃないとダメ、と思ってる

じゃないです。読みやすさと、何をしたい人かが伝わるかの方が大事なことが多いです。

毎週更新しないと意味がない、と思ってる

更新が少なくても、中身がちゃんと整っていれば十分アピールになるんじゃないでしょうか？乃木坂の吉田綾乃クリスティーなんて、アイドルなのに公式ブログは年に一回くらいしか更新しないこともある、くらいの話です。（普通は最低でも月一とか）

吉田綾乃クリスティー公式ブログの「NEW ENTRY」一覧（更新は年に数回程度の例）

SNS・GitHub・技術ブログとの違い

TwitterとかのSNS

流れていきやすく、長く残す・構造化するには向かないこともあります。

GitHub

コードやリポジトリの置き場として強い一方、文章や自己紹介、作品の並べ方はプロフィールだけだと足りないこともあるかな、と思います。

Qiita / Zenn など

記事単体は強いですが、自分全体のハブとしては、個人サイトがあるとまとまりやすいです。

個人サイトに、いろいろなページへのリンクをまとめておくと、役割が整理しやすい、って感じです。

作る前に決めておくとラクだったこと

サイトでなにを中心に置くか

作品一覧

画像やデモ、GitHub へのリンクを並べる中心。

ブログ・学習メモ

技術記事や日記に近いものを中心。

プロフィール

短文の自己紹介と、連絡先・リンク集を中心。

最初から全部やろうとしなくてよくて、ひとつに絞ると迷いが減りました。

履歴書・ES に載せるとき

技術の話は、履歴書に書く人向けの話も混ざってます。働く前提で書類に載せる人向けの話です。自分はまだここまでやってないです。

URL の書き方

「個人サイト: https://...」のように、何が見られるページかがわかると読みやすかったです。

書類と中身の整合

履歴書に Python と書いてあるのに、サイトに関連する記述がまったくない、みたいな食い違いは避けた方が無難です。技術アピールしたいなら、公開してるものと対応が取れると話しやすいです。

例:

  • 「制作物・学習記録をまとめた個人サイト:（URL）」

プライバシーと公開範囲

本名・顔写真

出すかどうかは人それぞれです。出さなくてもいいと思います。というか出してる人の方が少ないんじゃないかな。

連絡先

メールや SNS を載せる場合は、スパムや想定外の連絡も念頭に置いておくくらいでいいと思います。

続かないとき

ページを増やしすぎない

最初はトップだけ、でもありだと思います。 自分も最初はトップページだけでした。アドベントカレンダーをきっかけにだんだん記事が増えてきたので、記事のページとか作品のページも増やすようになりました。

完璧を待たない

後からいくらでも書き換えられるので、適当でいいから、もし作成してるものがあるのであればいまのまま出してもいいと思います。

お金と置き場の話（ざっくり）

お金の話だけ先にすると、作るだけならローカルで書いて終わりでもタダですし、公開まで無料でやるなら GitHub Pages みたいな無料枠もあります。独自ドメインを取るとか、有料プランのサービスを使うとか、そういうところから費用が出る、って感じです。

まとめ

就活やインターンでサイトを使うとき、世の中ではよくこう言われる、って話です（自分は働きたくない派ですが）。

書類とサイトの内容が矛盾してないか

書類に書いた内容と、サイトの中身が大きく食い違ってないか。

作品や学習が辿れるか

リンク切れに気づけるか、も含めて。

技術面でアピールしたいなら

書いたスキルと公開しているものがつながっているか。

自分が何をしたい人かが伝わるか

短時間で伝わるか。

URL がそのまま貼れるか

https か、公開範囲は意図どおりか、など。

最初は少なくてもいいので、どこかに出してから足していく、でいいと思います。

万能ってわけじゃないですが、説明したり制作したりするときに、ちょっと楽になることがある、くらいに思ってもらえたらいいです。

働きたい人も働きたくない人も、技術力を見せたい人も、何か作りたいのに続かない人も、自分のペースで試してみてもらえたらうれしいです。

というか皆さんサイト作りましょう。楽しみに待ってます。
